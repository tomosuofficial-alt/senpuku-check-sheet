対象: 仕込み型定番メニューの欠品防止（在庫が申告されなければ未対応として自動検知）

| ファイル | 内容 |

## 対象品目（2026-07-04 確定・使い分け方針）

「マニュアルの仕込み関連の追加項目」10品は以下のように使い分ける:

**A. 提供可能数の数値申告（STOCK品目）— 完成ストック5品**

| itemId | 品目 | 単位 | PAR |
|---|---|---|---|
| STOCK001 | ポテサラ | 食 | 空欄（ABC分析後に記入） |
| STOCK002 | サラダベース | 食 | 空欄 |
| STOCK003 | タルタルソース | 食 | 空欄 |
| STOCK004 | ポテサラソース | 食 | 空欄 |
| STOCK005 | ポーチドエッグ | 個 | 空欄 |

※ PAR空欄の間は「記録＋未申告検知」のみ稼働。閾値LINEアラートはJ列に数値を入れた時点で有効化。

**B. 仕込みの✓チェック項目 — 薬味系5品**（ばんネギ／白髪ネギ／ツマ／にんにくおろし／生姜おろし）

→ 開店項目の正本は**開店マニュアルのGoogleドキュメント**（`SYNC_DOCS['開店']`）。ドキュメントの「出勤」セクション（仕込みの近く）に5行追記して `syncCheckItemsFromDocs()` を実行すれば自動反映。**コード変更・シート直編集は不要**。行はそのまま項目名になる（例: `ばんネギの仕込み`）。

## 反映手順（3ステップ）

### 1. Code.gs を Apps Script に反映（必ず最初に）

1. スプレッドシート（`1SvnkJzDm6AzcyGHuJOUprppQWnSUUEcJUtv5HMhAuAk`）→ 拡張機能 → Apps Script
2. リポジトリの `Code.gs` 全文を貼り付けて保存
3. **デプロイ → デプロイを管理 → 既存デプロイを編集 → 新バージョン**で公開
   （「新しいデプロイ」にするとURLが変わりフロントが繋がらなくなるので注意）

### 2. 品目を投入

1. GASエディタで **`addStockItems()` を実行** → STOCK 5品がマスタに入る（再実行すると作り直し）
2. GASエディタで **`addPrepItems()` を実行** → 仕込み詳細10品（PREP001〜010）がマスタに入る
3. マスタの J1 に `par`、K1 に `unit` のヘッダーを入れる（推奨・任意）
4. 「チェック履歴」シートの H1 に `提供可能数` ヘッダーを入れる（推奨・任意）

※ 品目・PARの変更はシートの STOCK/PREP 行を直接編集するだけ。コード修正不要。

#### 仕込みプルダウン（PREP）について
- フロントは itemId が `PREP` 接頭辞の項目を、開店タブの「仕込み」行の下に**アコーディオン（プルダウン）**でまとめて表示する（既定は閉、ヘッダーに「n/10 完了」）。
- 各詳細品目は**個別にチェック→履歴に記録**される（通常のチェック項目と同じ経路。GASロジックの追加なし）。
- 10品すべて完了で親「仕込み」(KAI002)も自動的に完了扱いになり、開店チェックの完了判定・LINE通知と整合する。
- PREP行は `syncCheckItemsFromDocs()`（開店/閉店のドキュメント同期）の削除対象から除外済み（STOCKと同様）。**新Code.gsを反映してから同期する**こと。
- 「仕込み」の詳細10品のうち5品（ポテサラ/サラダベース/タルタルソース/ポテサラソース/ポーチドエッグ）は、別途「提供可能数」の数値申告（STOCK）にも登場する。**仕込みの✓＝作ったか、提供可能数＝何食出せるか**で目的が異なるため両立させている。

### 3. 動作確認

1. アプリの開店準備タブに「提供可能数」セクションが出ること
2. PAR以下の数値（例: J=5 の品目に 3）を入れて送信 → 代表LINEに「📦 提供可能数アラート」が届くこと
3. 未申告のまま18:00を過ぎる → 既存の未完了通知に「📦 提供可能数が未申告: ◯◯」が含まれること
4. 履歴モーダル（📋）の開店カードに「本日の提供可能数」一覧が出ること

## 仕様メモ

- **0食も有効な申告**（=売り切れ報告。履歴では「売切」表示・アラート対象）
- 申告済みでも入力欄はロックされない（在庫が減ったら上書き申告できる。最新値が有効）
- LINE通知の宛先・トリガー（18:00/22:30/7:00）は既存のまま。新規トリガー設定は不要
- 通知判定は「チェックを付けたか」ではなく**H列に数量が記録されたか**で行う（"やった風"防止）
- 開店/閉店の項目は `syncCheckItemsFromDocs()`（Googleドキュメント→マスタ同期・手動実行）で作り直されるが、**STOCK行は同期の削除対象から除外済み**なので消えない（`deleteRowsByCategory_` 参照）。ただし**必ず新しい Code.gs を反映してから同期を実行する**こと（旧コードで同期するとSTOCK行が消える）
